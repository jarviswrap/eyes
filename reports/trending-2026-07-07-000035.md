**💡 功能概述**: herdr 是一个终端内运行的代理多路复用器，核心功能是让用户在一个终端窗口中同时运行、监控和管理多个编码代理（如 AI 代码生成代理）。每个代理获得一个真实的终端会话，支持分屏、标签页和工作区，并通过侧边栏实时显示每个代理的状态（阻塞、工作中、完成、空闲）。它还提供持久化会话支持（分离后进程不终止）、跨终端重新附加、鼠标原生交互以及脚本化的 Socket API 和 CLI，方便代理自身或外部工具驱动。面向的用户群体包括：使用多个 AI 编码代理的开发者、需要同时管理多个终端会话的运维人员，以及希望在 SSH 环境或手机终端上管理代理的高级用户。主要使用场景包括：同时运行多个代码生成或重构任务、观察代理进展并切换关注、在无图形界面的服务器上管理代理工作流。

**⚙️ 技术栈**: 该项目使用 Rust 语言编写，核心特点包括：1）利用 Rust 的性能和跨平台能力构建单一约 10MB 的静态链接二进制文件，无需外部依赖；2）后台服务器架构实现会话持久化和分离/重新附加功能；3）基于终端绘制（如 ratatui、crossterm 等）实现鼠标原生支持、分屏和状态栏渲染；4）通过 Unix 域套接字或 WebSocket 提供本地 Socket API，供代理和外部程序驱动；5）采用事件循环和异步 I/O（可能基于 tokio）处理多个代理的并行输出和状态更新；6）插件系统支持任意语言编写的扩展。架构设计上采用 C/S 模式：后台守护进程管理所有代理会话和状态，前端客户端（CLI 或终端 UI）连接并显示视图，分离后客户端可退出而代理继续运行，再次连接时恢复界面。

**🎯 痛点解决**: 1. 多代理管理混乱：使用 AI 编码代理时，多个代理同时运行导致终端窗口杂乱无章，难以快速了解每个代理的进度。2. 缺乏代理状态可视化：传统终端复用器（如 tmux）只提供窗口布局，无法自动识别代理是阻塞、工作中还是完成，需要用户手动切换查看。3. 分离后进程中断：许多终端工具在用户断开 SSH 或关闭终端后退出进程，导致长时间运行的任务丢失。4. 依赖图形界面的限制：GUI 管理器（如 iTerm2 分屏）无法在纯终端环境或 SSH 连接下使用，且往往需要 Electron 等重框架，资源消耗大。5. 难以编程化集成：现有方案缺少统一的 API 供代理或 CI/CD 系统调用，进行自动化编排。用户选择 herdr 是因为它同时解决了以上痛点：轻量级单二进制、实时状态概览、持久会话、鼠标原生、可脚本化。

**🏆 竞品分析**: 同领域知名项目包括：1）tmux：最流行的终端复用器，支持持久会话、分屏和分离重附，但缺乏代理感知和状态可视化。2）screen：类似 tmux 的经典工具，功能更基础。3）dvtm / abduco：动态窗口管理器组合，轻量但用户界面较原始。4）Terminator / Tilix：基于 GUI 的终端分屏工具，依赖图形桌面且不支持持久会话。5）图形化代理管理工具（如 Claude Code 提供的活动管理器）：仅限于特定代理平台，不通用。herdr 的差异化优势：① 原生代理意识——自动识别并显示每个代理的阻塞/工作/完成状态，无需配置 hooks；② 真实终端视图——每个代理拥有完整终端，全屏 TUI 也能正常渲染，而非模拟视图；③ 零配置——开箱即看状态，无需学习 tmux 键绑定；④ 超轻量——单一 10MB Rust 二进制，无 Electron、无 GUI，可在任何终端甚至手机 SSH 中运行；⑤ 脚本化——提供 Socket API 和 CLI，代理可以自行驱动布局和状态更新，适合 AI 工作流编排；⑥ 鼠标原生——支持点击、拖拽分屏，降低终端操作门槛。

**⚙️ 技术栈**: 项目主语言为 TypeScript，桌面端可能基于 Electron 或 Tauri（结合 Ghostty 类终端，暗示使用 WebGL 渲染）。前端可能采用 React 或 Vue 等现代框架。移动端有 iOS（App Store）和 Android（APK）原生应用。关键架构特点：利用 git worktree 作为隔离机制，允许多个 agent 共享同一仓库但互不干扰；通过中央调度器（Orchestrator）统一管理 agent 生命周期、会话同步与结果收集；通信层面可能通过 WebSocket 或类似协议实现桌面与移动端的实时同步。终端核心采用高性能 WebGL 渲染引擎，支持无限分屏和复杂 UI 交互。整体设计强调模块化、可插拔的 agent 适配器，以支持多种第三方 AI 编码终端（如 Codex CLI、Claude Code 等）。

**🎯 痛点解决**: 1. 多 agent 管理混乱：开发者同时使用不同 agent（如 Claude Code 和 Codex）时缺乏统一界面，Orca 集中管理并隔离工作区。2. 结果对比困难：手动运行多个 agent 并比较输出耗时且易出错，Orca 提供并行 worktree 和一键对比/合并功能。3. 远程监控缺失：长时间运行 agent 时无法随时查看进度，移动端 Companion 允许用户从手机接收通知并发送后续指令。4. 环境冲突风险：不同 agent 可能修改同一文件，Orca 通过隔离 worktree 彻底避免冲突。5. 学习成本高：需要分别配置和管理多个 agent，Orca 提供统一入口和配置，降低使用门槛。

**🏆 竞品分析**: 同领域知名项目包括：Cursor（AI 原生 IDE，集成多模型但有专有环境）、OpenAI Codex CLI（官方命令行工具）、Anthropic Claude Code（独立终端应用）、OpenCode（开源编码 agent）、以及多 agent 编排框架如 AutoGen、CrewAI。Orca 的差异化优势：1. 并行与隔离：允许同时运行多个不同 agent 且每个在独立 git worktree，这是大多数产品不具备的。2. 移动端支持：提供原生手机 App 进行远程监控和交互，领先于纯桌面工具。3. 终端体验：采用高渲染性能的 WebGL 终端，支持无限分屏，优于标准终端。4. 跨平台兼容：Windows/macOS/Linux 桌面 + iOS/Android 移动端。5. 开放生态：不锁定特定 agent，允许用户使用自带订阅的任意编码 agent（如各自 API 密钥），而 Cursor 等专有产品限制 agent 选择。

## #20 ChromeDevTools/chrome-devtools-mcp

- **总 Stars**: 46,087 | **本周 Stars**: +1,394 | **Forks**: 2,999 | **语言**: TypeScript
- **描述**: Chrome DevTools for coding agents
- **GitHub**: [ChromeDevTools/chrome-devtools-mcp](https://github.com/ChromeDevTools/chrome-devtools-mcp)
- **近一周上榜**: 1 天

**💡 功能概述**: 该项目实现了一个基于 Model-Context-Protocol (MCP) 的服务器，让 AI 编码助手（如 Antigravity、Claude、Cursor、Copilot）能够直接控制和检查实时 Chrome 浏览器。核心功能包括：通过 Chrome DevTools 获取性能跟踪和可操作见解、分析网络请求、截取屏幕截图、检查控制台消息（支持源映射堆栈跟踪），以及使用 Puppeteer 实现可靠的自动化操作。面向用户群体是使用 AI 编码助手的开发者，主要使用场景包括：自动化调试和测试、性能分析、浏览器行为检查，以及让 AI 代理与 Web 应用进行交互。

**⚙️ 技术栈**: 主要使用 TypeScript 开发，基于 Puppeteer 与 Chrome 浏览器进行自动化交互，集成 Chrome DevTools Frontend 获取调试和性能数据。架构上采用 MCP（模型上下文协议）作为 AI 代理与浏览器之间的通信桥梁，将 DevTools 功能封装为可调用的工具（tools）。服务器以 NPM 包形式发布，支持 CLI 直接使用或不通过 MCP 运行。还利用了 Google CrUX API 获取真实的用户体验数据以补充性能分析。

**🎯 痛点解决**: 解决了 AI 编码代理无法直接与真实浏览器交互、无法获取深度调试信息（如性能跟踪、控制台错误、DOM 状态）的痛点。传统浏览器自动化库（如 Playwright/Puppeteer）需要开发者编写脚本，而该项目允许 AI 代理通过自然语言调用标准化工具，降低了自动化门槛。同时，它提供了开箱即用的 Chrome DevTools 能力（性能分析、网络记录、元素检查），避免了开发者手动打开 DevTools 进行排查的繁琐过程，适合快速迭代和调试场景。

**🏆 竞品分析**: 同类项目包括：Puppeteer（Google 的浏览器自动化库）、Playwright（Microsoft 的跨浏览器自动化框架）、Selenium（Web 自动化测试工具）、Browserbase（用于 AI 代理的云浏览器 MCP 服务）、以及其他针对 LLM 的浏览器控制工具（如 Browserless、Playwright-MCP）。该项目的差异化优势在于：1）原生集成 Chrome DevTools 而非仅提供基础自动化，能获取性能跟踪、源映射堆栈等深度调试信息；2）专为 MCP 协议设计，与主流 AI 助手（Claude、Cursor、Copilot）即开即用；3）由 Chrome DevTools 团队官方维护，质量与更新有保障；4）注重隐私与数据收集，提供退出选项；5）支持 CLI 模式灵活使用，不强制依赖 MCP。
